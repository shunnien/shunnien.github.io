程式設計的基本認知

發表於 2017-08-11 更新於 2019-06-14 分類於 程式技術筆記 閱讀次數： Disqus：

在程式開發上，應該都需要降低耦合，增強內聚這樣的認知，而為了達成這樣的目標，就會有介面、原則等等…

抽象

找出關鍵性特徵並加以描述；抽象表示一個群集，所以應該依賴抽象，而不是倚賴具體。倚賴抽象的目的是希望程式碼易於變動。

目標

  • 高內聚
    內聚性除了程式上之外，還有設計上的內聚；設計上的內聚，就是命名上的設計，例如：Math 這 Class 的方法都是低內聚，裡面的方法，看到 Math 就知道裡面的方法都是數學相關的運算，不會再 Math 裡面看到 sql connection 的方法，因為這完全不相關。
  • 低耦合

特性

  • 繼承
    繼承是為了多型的目的；B 繼承 A，所以 B 就是 A。
  • 封裝
  • 多型
    有泛型 、多載也是多型、強制同型也是多型(編譯器比較多使用)

原則

  • 單一職責原則(SRP)
  • 里氏替換原則(LSP)
    繼承父類別後，不要遮蔽父類別的方法。
    簡單說，使用父類別的地方，都可以使用子類別替換。
  • 倚賴倒置原則(DIP)
    倚賴抽象
  • 介面隔離原則(ISP)
  • 開閉原則(OCP)
    對擴展開放，對修改封閉。
  • 最少知識原則(迪米特法則LOD)
